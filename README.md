#CS-320 Portfolio

## Reflection:

1. To make sure my code or software is functional and secure, I focus a lot on testing and validation.
I use unit tests to check that each part of the program works the way it is supposed to, and I try different inputs to catch possible errors.
I also make sure to validate user input so the program doesn’t crash or accept bad data.
Security is important too, so I try to follow good coding practices like not hardcoding sensitive information and handling exceptions properly.
Overall, testing and careful coding help make the program more reliable.

2. When it comes to interpreting user needs, I start by carefully reading the requirements and breaking them down into smaller parts.
I think about what the user is actually trying to accomplish and what features the program needs to support that.
Then I translate those needs into code by creating classes and methods that match the requirements.
I also check my work against the instructions to make sure I didn’t miss anything.
Testing again helps here because it shows whether the program is doing what the user expects.

3. For designing software, I like to plan things out before I start coding.
I think about the structure of the program, such as what classes I need and how they will interact.
I try to keep my design simple and organized so it’s easier to understand and debug later.
I usually build the program step by step, testing as I go, instead of trying to do everything at once.
If something doesn’t work, I go back and adjust the design.
This approach helps me stay organized and create code that is easier to maintain.
